react的指向

Created2021-07-10|Updated2022-04-25
|Post View:

介绍

​ 声明式javascript库

​ 被视为JS一等公民中的对象，因此可以React元素在应用程序中当作参数来传递。在react中，我们还可以使用react元素的数组来渲染多个元素。

​ 流程：react jsx->render函数->生成Vnode->当有变化时，新老Vnode diff ->diff 算法对比，并真正去更新真实DOM。

特点

  • 数据不可变，需要setstate驱动新的state替换老的State。

  • 当数据改变时，以组件为根目录，默认全部重新渲染

  • 生命周期

  • 单向数据流：props，主要通过onchage/setstate的形式更新数据（受控组件！！）

  • 组件通信：props,context,redux

  • 组件嵌套:props.children

  • class组件下的this是不存在的，需要手动为方法绑定this才能在回调函数里使用方法。

    方法一：在构造函数里使用this.fun=this.fun.bind(this)
    方法二：在回调函数里使用时直接用this.fun.bind(this)
    方法三：使用箭头函数：（）=>this.fun()箭头函数会继承父作用域的this

生命周期

  1. render(),当render被调用时，它会检查this.props和this.state的变化并返回要渲染的数据类型
  2. constructor()，如果不初始化state或不进行方法绑定，则不需要为react组件实现构造函数

组件编写方式

class

创建一个class类并且让他继承React.Componet，同时通过构建函数处理props，将props挂载到super上，并且声明state状态初始化一个对象，可以添加一些自定义方法，在Render方法中return出JSX的模板。

function

函数组件，组件中只包含一个render方法。不需要定义一个继承于React.Componet的类，直接定义函数，这个函数接收一些参数，然后返回需要渲染的元素。

组件传值

1.在父组件使用子组件的时候绑定定义一个方法,使用这个方法的时候传递参数，在函数里面通过value{sth}接收参数,然后在子组件中通过this.props.value接受参数。

2.在需要同时获取多个子组件数据，或者两个组件之间需要相互通讯的情况是，需要把子组件的state数据提升至其共同的父组件当中保存。之后父组件可以通过props价格状态数据传递到子组件当中。这样应用当中所有组件的状态数据就能够更方便地同步共享了。

参数

state

在react组件的构造函数中设置this.state来初始化state。改变state里面的值来设置不同的状态。

state对于每个组件来说是私有的，因此不能直接通过其余组件来改变目标组件的state

setState

每次在组件中调用setState时，react都会自动更新其子组件，setstate它用来改变state里面的值。

super

在javaScript class中，每次定义其子类的构造函数时都需要调用super方法，所以在所有含有构造函数的Reat组件中，构造函数必须以super(props)开头

on[event]

在react中，有一个命名规范，通常会将待变事件的监听prop命名魏on[event]，将处理事件的监听方法命名为handle[event]

不可变性与可变性

改变数据有两种方式：直接修改变量的值/使用新的一份·数据替换旧数据。

不直接修改数据的好处：

1.简化复杂的功能

2.跟踪数据的改变

3.确定在react中何时渲染（帮助在React中创建Pure Componets，可以更轻松的确定不可变数据是否发生了改变从而而确定合适对组件进行重新渲染）

组件插槽

key

当我们需要渲染一个列表的时候，react会存储这个列表每一项的相关信息。当我们要更新这个列表时，react需要确定哪些项发生了改变。key属性用来区分不同的列表项和他们的同级兄弟列表项。每当一个列表重新渲染是，react会根据每一项列表元素的Key来检索上一次渲染时与每个Key所匹配的列表项。如果react发现当前的列表有一个之前不存在的key，那么就会创建出一个新的组件。如果react发下和之前对比少了一个key，那么就会销毁之前对应的组件。如果一个组件的key发生了变化，这个组件会被销毁，然后使用心得state重新创建一份。

当react圆度被创建出来的hi后，react会提取出key属性，然后把key直接存储在返回的元素上。key不能通过this.props.key来获取，react会通过key来自动判断哪些组件需要更新。组件是不能访问到他的key的。

每次构建动态列表时，都建议指定一个合适的key。

如果没有指定key，react会发出警告并把数组的索引当作默认的key，但是此时如果想要对列表进行重新排序，新增，删除操作，是会存在问题的，但是如果不需要这些操作就可以使用数组索引。

组件的key并不需要在全局保证唯一，只需要在当前的同一级元素之前保证唯一就可以了
