总结一下项目操作（git）

开发前创建开发分支

$ git checkout master 切换到分支master

$ git pull 取回远程主机某个分支的更新，再与本地的指定分支合并，完整为：git pull [options] [<repository> [<refspec>…]]在这里就是保证本地的分支是最新的代码

$ git checkout -b feature-wdy-featureName 创建并切换到分支 feature-wdy-featureName

代码写好了之后或者说你想要推送到gitlab上面就

git add .

git commit -m

git push 三连

PS：推送代码到并创建远程分支

在修改代码的时候有意见严肃的事情要说

最好在feature分支（你的开发分支）上的时候写，因为再哪个分支上写，代码就是哪个分支的，写完之后可以上传。然后再切到其他分支。不然你可能就完蛋了。因为切到的哪个分支没有你在这个feanture分支上的功能嘿！

写好代码之后add，commit，push三连！

其次就是要在其他大分支上（test/master）合自己的代码，而不是在自己的分支上来和别人的代码，因为你写的只有一点点代码，但是大分支上不知道有多少修改或新增的代码，用脚脖子想用谁合谁会比较容易！！

具体合代码的操作（以test为例）

虽然gitlab上有可视化合代码，但是显然熟知git指令会让操作更简单易懂
  1. 在你的开发分支上保存并推送你的代码到远程
  2. 切换到你要合代码的分支上，开始git merge 你的开发分支 ，然后开始解决合并冲突（如果这个时候你的vscode上面有Git History， GitLens这两个插件的话，会帮助你识别冲突的代码是谁以及什么时候写的）
  3. 然后上传到test分支上面

创建MR

从master分支，切出MR（merge request）的目标分支

$git checkout master

$git pull

$git status 查看当前分支

$git push --set-upstream origin release--wdy-featureName 注意名字前面的前缀

创建MR，在可视化页面创建MR，主要是为了让reviewer看看代码，方便自己修改

其实可以直接用git命令合的，具体参考前面合代码的操作

删除多余的commit（这个操作我还没有成功过55555，不要轻易用这个）

$git log查看所有的commit并记住他们的唯一号码

$git rebase -i ID

进入页面后，输入i进入编辑模式，ESC 退出编辑模式，ZZ两个大写ZZ保存文件并退出。

$ git log 再查看一下

上线的操作

  1. 首先，切到pre_master分支，（在切换到另一个分支前记得保存在当前分支上的操作哦！）

  2. git pull 保证分支是最新的

  3. 在pre_master上合自己的代码并git commit

  4. 使用指令 npm version patch生成并返回新的tag

  5. git push --tags 给项打上新的tag并提交

  6. 在gitlab上CI/CD板块查看版本运行情况

  7. 成功的话就可以进行下一步操作

    成功辣，然后闪电会发给你通知

    然后你可以复制构建tag到你的闪电工单上，闪电工单就是一个

    ·https://shandian.xiaoduoai.com/·

    链接在这了（如果登不上去就找运维帮你开权限，运维就是前端组里公告上挂着的老师。），上线的话，需要在CD管理里面创建一个工单，然后就是各种填信息，最终要的就是要把tag填进去啦，然后发起审批，审批好了，大家（主要是测试同学哈）说可以发布了，那就发布工单——测试回测到预发布环境——发布工单——测试再测·辣。

    关于回退

    git show 要回退到的版本号（v1.1.139）查看他的唯一标识，

    git reset --hard 标识（8d5689733361baf11685ef893f41dce964465d5d）

    git log 查看版本日志

    git push -f 保证远程·1仓库与本地一直

    然后你的版本应该已经回退到你需要的版本了。

其余常用命令（有待补充）

$ git branch | grep wdy 查看含有“wdy”的分支

$git reflog --date=local feacture-wdy-contractExoire（BranchName） 查看当前分支是从哪个分支切下来的

$git branch -d Name 删除本地分支

$git push origin --delete name 删除远程分支

git branch -D pre_master 删除本地分支

git branch pre_master origin/pre_master 删除远程pre_master分支?

git checkout pre_master 再直接切到pre_master

git tag -d tag名（eg：v1.1.140）删除本地tag

git show 查看tag

git push origin :refs/tags/v1.1.141 删除远程仓库里的tag，但是你可能会没有权限删除。

一个拉取分支操作

git revert 和 git reset

git revert 撤销某次操作，此次操作之前和之后的commit和history都会保留，并把这次撤销作为一次最新的提交。

git revert commit (版本号) 撤销指定的版本，撤销也会作为一次提交作为保存。，该指令是提交一个新的版本，加ing需要revert版本的内容在反向修改回去，版本会递增，不影响之前提交的内容。

git reset 直接删除指定的commit

常用回退操作：

回退到某一版本：git reset –hard commit _id

强制推送到远程仓库：git push origin -f
